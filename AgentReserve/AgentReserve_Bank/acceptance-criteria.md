# AgentBank 黑客松验收标准

> **验收哲学**: 可量化、可检查、一分钟内能验证通过/失败

---

## 一、GitHub 仓库验收标准

### 1.1 基础完整性 [必须]

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 仓库可访问 | 公开仓库，URL可直接访问，无需登录 | 浏览器无痕模式打开仓库URL |
| 目录结构 | 包含 `agentbank/`、`data/`、`static/`、`demo/`、`README.md`、`requirements.txt` | 查看仓库根目录 |
| 代码可运行 | `main.py` 存在且包含FastAPI应用实例 | 阅读代码确认 |
| 最近提交 | 最后一次提交在黑客松截止前（2026-05-03 08:00前） | 查看commit时间 |
| 提交历史 | 至少有5次有意义的commit（非单次提交所有文件） | `git log` 查看 |

### 1.2 代码质量 [尽量]

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 核心API数量 | 至少实现8个端点（见开发计划） | 数`@app.get/post`装饰器数量 |
| 数据模型 | 使用Pydantic定义核心模型（Agent, Loan, TokenPool等） | 查看`models.py` |
| 错误处理 | 关键API有try-catch，返回HTTP状态码合理（400/404/500） | 用错误参数调API |
| 代码注释 | 核心函数有docstring | 抽查3个函数 |

### 1.3 仓库评分标准

- **通过（✅）**: 基础完整性全部满足
- **优秀（⭐）**: 基础完整性 + 代码质量全部满足
- **不通过（❌）**: 基础完整性任一项不满足

## 二、服务部署验收标准

### 2.1 可访问性 [必须]

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 公开URL | 提供HTTPS公开URL（Render/Railway等），非localhost | 浏览器直接访问 |
| 健康检查 | `GET /health` 返回 `{"status":"ok"}`（HTTP 200） | `curl` 或浏览器访问 |
| 服务稳定 | 连续访问3次 `/health`，全部200 | 连续刷新3次 |
| 响应时间 | `/health` 响应时间 < 5秒 | 浏览器Network面板或curl -w %{time_total} |

### 2.2 API可用性 [必须]

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| Swagger UI | `/docs` 页面可加载，显示所有API端点 | 浏览器访问 `/docs` |
| 核心API-1 | `POST /agents/register` 可用，返回Agent信息 | Swagger UI中Try it out |
| 核心API-2 | `POST /lending/borrow` 可用，返回Loan信息 | Swagger UI中Try it out |
| 核心API-3 | `POST /lending/repay` 可用，可成功还款 | Swagger UI中Try it out |
| 核心API-4 | `POST /estimator/predict` 可用，返回预估结果 | Swagger UI中Try it out |
| 查询API | `GET /pool/status` 返回流动性池数据 | Swagger UI中Try it out |

### 2.3 Dashboard页面 [尽量]

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 页面可访问 | 访问根路径`/`或`/dashboard.html`返回可视化页面 | 浏览器访问 |
| 数据展示 | 页面展示至少3项数据（总流动性、活跃Agent数、Token类型分布） | 肉眼检查 |
| 页面美观 | 使用CSS框架（Tailwind/Bootstrap）或自定义样式，非纯文本 | 肉眼检查 |

### 2.4 部署评分标准

- **通过（✅）**: 可访问性 + API可用性全部满足（至少6个API可用）
- **优秀（⭐）**: 以上全部 + Dashboard页面可访问且美观
- **不通过（❌）**: 可访问性任一项不满足，或核心API少于5个可用

## 三、Demo Case 验收标准

### 3.1 Demo Case 1：应急借贷 [必须]

**场景描述**: Agent A在执行任务时Token不足，向AgentBank紧急借Token，完成任务后归还。

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 故事完整 | Demo展示完整的"借→用→还"闭环 | 观看录屏/现场演示 |
| Agent注册 | 展示Agent A的注册过程（或已预注册） | 录屏中有`/agents/register`调用 |
| 借款操作 | 调用`/lending/borrow`成功，返回loan_id和利率 | Swagger UI中显示响应 |
| 余额变化 | 借款后Agent A的可用额度/余额发生变化 | 调用`/agents/{id}/balance`展示 |
| 还款操作 | 调用`/lending/repay`成功，返回repaid信息 | Swagger UI中显示响应 |
| 状态更新 | 还款后Loan状态变为"repaid" | 查询loan详情确认 |

### 3.2 Demo Case 2：预估+智能借贷 [必须]

**场景描述**: Agent B在接任务前，先用预估服务计算Token需求，然后根据预估结果申请借款。

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 预估调用 | 调用`/estimator/predict`，输入任务描述 | Swagger UI中展示请求体 |
| 预估结果 | 返回estimated_tokens（数值>0）、recommended_borrow（数值>0） | Swagger UI中展示响应 |
| 智能借款 | 根据预估的recommended_borrow申请借款 | 录屏中展示借款金额与预估关联 |
| 信用影响 | 如果实现信用评级，展示信用好的Agent利率更低 | 对比两个Agent的借款利率 |

### 3.3 Demo 呈现形式 [必须]

| 检查项 | 验收标准 | 检查方法 |
|--------|----------|----------|
| 录屏或现场 | 提供录屏视频（MP4/GIF）或现场可演示 | 播放视频/现场操作 |
| 时长 | 单个Demo时长1-3分钟，总Demo时长不超过6分钟 | 计时 |
| 清晰度 | 能清楚看到API请求/响应的JSON内容 | 全屏播放检查 |

### 3.4 Demo评分标准

- **通过（✅）**: 2个Demo Case全部展示，每个Case的借款+还款流程完整
- **优秀（⭐）**: 以上全部 + 预估服务结果合理 + 信用评级展示利率差异
- **不通过（❌）**: 少于2个Demo，或任一Demo的借贷流程不完整

## 七、综合验收总表

### 7.1 一票否决项
以下任一项不满足，直接判定为不通过：

- [ ] GitHub仓库不是公开的
- [ ] 没有可访问的公开URL
- [ ] `/health`端点无法返回200
- [ ] 少于2个可运行的Demo Case
- [ ] README缺失项目概述和安装指南

### 7.2 核心评分表

| 交付物 | 权重 | 通过标准 | 实际得分 |
|--------|------|----------|----------|
| GitHub仓库 | 15% | 基础完整性全部满足 | ⬜ |
| 服务部署 | 25% | 公开URL + 核心API可用 | ⬜ |
| Demo Case | 30% | 2个Case流程完整 | ⬜ |
| README | 15% | 内容完整性全部满足 | ⬜ |
| PPT/展示 | 10% | 5页+，结构完整 | ⬜ |
| 小红书视频 | 5% | 发布成功，展示功能 | ⬜ |

**总分计算**: 各交付物实际得分 × 权重之和。
- **≥80分**: 优秀
- **≥60分**: 通过
- **<60分**: 需补交

### 7.3 最终检查流程（截止前1小时执行）

```
□ 打开GitHub仓库，确认公开
□ 点击README中的部署URL，确认可访问
□ 访问 /health，确认返回200
□ 访问 /docs，确认Swagger UI加载
□ 在Swagger UI中执行：注册Agent → 借款 → 查余额 → 还款
□ 访问 / 或 /dashboard.html，确认页面有数据
□ 播放Demo录屏，确认2个Case都完整
□ 打开PPT，确认页数和内容结构
□ 检查README中的链接全部可点击
```

## 八、验收工具速查

```bash
# 1. 健康检查
curl https://your-url.onrender.com/health

# 2. 注册Agent
curl -X POST https://your-url.onrender.com/agents/register \
  -H "Content-Type: application/json" \
  -d '{"id":"agent_001","name":"DemoAgent"}'

# 3. 借款
curl -X POST https://your-url.onrender.com/lending/borrow \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"agent_001","token_type":"claude","amount":1000,"purpose":"翻译任务","duration_hours":24}'

# 4. 查余额
curl https://your-url.onrender.com/agents/agent_001/balance

# 5. 还款
curl -X POST https://your-url.onrender.com/lending/repay \
  -H "Content-Type: application/json" \
  -d '{"loan_id":"loan_xxx","agent_id":"agent_001","amount":1050}'

# 6. 预估
curl -X POST https://your-url.onrender.com/estimator/predict \
  -H "Content-Type: application/json" \
  -d '{"task_description":"将一篇5000字的中文论文翻译成英文"}'

# 7. 流动性池
curl https://your-url.onrender.com/pool/status
```
